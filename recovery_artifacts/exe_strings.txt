OVERVIEW
--------
The C/stdin port has no embedded script runtime. Mods are pure data: text files on
disk, merged into RAM at startup (or after "mods reload"). This is ideal
for episodic DLC, seasonal events, translation packs, and licensed add-ons:
ship a folder users drop next to their save, or point AETER_MODS at your
install directory.
PLACEHOLDERS (character/*.txt overlays)
----------------------------------------
Use %NAME% %RACE% %CLASS% %ROLE% %PRONOUNS% in character overlay files;
they expand when the player opens that panel (same tokens as npcs/*.greeting.txt).
With a current location, the engine also fills %ROOMTITLE% %ROOMSLUG% %ROOM%
(slug) and %REGION%. From the world clock: %TIME% %PERIOD% %DAY% %SEASON%
%WEATHER% %TEMPC% (Celsius integer).
From play state: %COINS% %HP% %MAXHP% %SCORE% %TURNS% %INVCOUNT%
%INVMAX% %PACK% (e.g. 3/128) %READIED%.
World progress: %WORLDROOMS% %VISITED% %NOTECOUNT% %EXPLORE% (percent).
Character sheet fields: %STR% %AGI% %INT% %WIS% %CHA% %COR% %AGE% %GENDER%
%CLASSID% %RACEID% %BUILD% %MUSCLE%.
Derived profile fields: %POWER% %RESOLVE% %CUNNING% %PRESENCE% %HPPCT%
%RISK% %RISKLABEL% %TEMPER% %ARCHETYPE% %CORTIER% (alias %CORSTATE%).
Narrative-tone helpers: %VOICESTYLE% %MORALVECTOR% %THREATPOSTURE%.
Scene/state helpers: %SCENETONE% %TRAVELMOOD% %SOCIALSTANCE%.
NPC-memory helpers: %NPCHERE% %NPCDISPLAY% %NPCROLE% %NPCDANGER% %NPCTRUST% %NPCLEVERAGE% %NPCPRESENCE%
%LASTNPCHERE% %LASTNPCDISPLAY% %LASTNPCROLE% %LASTNPCATTITUDE% %LASTNPCDANGER% %LASTNPCTRUST% %LASTNPCLEVERAGE%.
Session: %LASTNPC% %LASTFOCUS% %LASTTOPIC% %LASTTOPICMOOD% %TOPICHEAT% %ROOMMODE% (verbose|brief).

MODS ROOT
---------
Default: same directory as your quicksave file, subfolder "mods".
Example: if save is  C:\Games\aeternitas64_save.txt  then mods go in
         C:\Games\mods\
Override (first match wins): in-game  mods directory <path>  |  --mods <dir>
  |  AETER_MODS=
  |  (Windows)  --pick-mods  at launch for a folder dialog.
Same idea for saves:  --save <path>  or  --pick-save  (Windows).
LOAD ORDER & DLC PRIORITY (manifest.txt)
----------------------------------------
Packs load in ascending order of manifest "priority=" (lower first),
then alphabetically by folder name. The LAST pack to touch a slug/id WINS.
  manifest.txt example:
    # Episode II drop 
 higher priority loads later and wins.
    priority=50
    id=my_episode_2
    title=Episode II 
 The Hollowridge Payoff
  Temporarily disable a pack without deleting it:
    disabled=1
    # or: enabled=0
priority= sets sort order; id/title are notes. Default priority 0.
  Strategy for content drops:
    
 Base / compatibility patch: priority=-100 or 0, early name e.g.
      010_story_patch
    
 Episode DLC: priority=40, folder 040_episode2
    
 Hotfix that must win everything: priority=90, folder 99z_hotfix
    
 Tie-break: sort by folder name (later in ASCII order loads later).
ROOM SLUGS & ITEM IDS
---------------------
Room file names must match internal slugs exactly, e.g. west_of_house.txt
Item files use the same ids the engine uses (mailbox, leaflet, lockpick,
scrap_metal, 
). Wrong spelling = silently ignored for that asset.
NPC entity slugs match room "entity" data, e.g. miller, blacksmith.
TOPIC FILES (npcs)
-----------------
  npcs/miller__rumor.txt  
 keyword "rumor" (lowercased in file name)
Player:  talk about rumor   
 if "rumor" appears in the phrase, match.
If several keywords match, the LONGEST keyword wins (DLC can override
generic topics). Mod topics are checked BEFORE built-in dialogue tables.
GREETINGS
---------
  npcs/miller.greeting.txt  replaces the stock greeting when the NPC
would say their greeting line (mod text wins if present).

SAMPLE PACK
-----------

that folder. Delete the folder to remove it; it will not overwrite edits

TESTING A DLC DROP
------------------
  1. Build your pack folder with manifest priority higher than base.
  2. Place it under mods/ (or AETER_MODS).
  3. Run the game or type  mods reload.
  4. Visit rooms you patched; examine items; talk with keywords.
  5. Ship updates by shipping a new folder with a higher priority or
     a later sort name 
 never require replacing the .exe.
LIMITATIONS (CURRENT BUILD)
---------------------------
 Text files: UTF-8 BOM and stray CR (Windows newlines) are normalized.
 No new rooms, items, or NPCs 
 only text overlays on existing data.
 No hooks into combat, merchants, or quest state 
 design fiction as
    if this were a localization / director's-cut text layer.
 Very large blurbs are capped per internal reader limits (~256KB).
